# CQ 量化精度问题修复 - 中文说明

## 🎯 问题总结

**症状**：运行 CQ 量化评测后，精度与 baseline 完全一致，量化似乎没有生效。

**根本原因**：在 loglikelihood 评估时，模型调用没有启用 KV cache，导致量化的 codebook 虽然被加载，但从未被实际使用。

**修复方式**：仅需修改 1 行代码，在 `lm_eval/models/huggingface.py` 第 993 行添加 `use_cache=True` 参数。

## 📋 详细诊断

### 工作流程回顾

1. ✅ **Fisher 梯度收集** - 正常工作
2. ✅ **Codebook 训练** - 正常工作  
3. ✅ **Codebook 加载** - 正常工作（`enable_cq_kv_cache` 正确加载）
4. ❌ **实际使用** - **这里出了问题！**

### 为什么量化没有生效？

CQ 量化的激活条件（`lm_eval/quantization/cq_cache.py` 第 320-328 行）：

```python
def patched_forward(self, *args, **kwargs):
    use_cache = kwargs.get("use_cache", None)
    past_key_values = kwargs.get("past_key_values", None)
    if getattr(self, "_cq_enabled", False):
        should_use_cache = use_cache if use_cache is not None else True
        if should_use_cache and past_key_values is None:
            kwargs["past_key_values"] = self._cq_cache_factory()  # 只有这里才创建量化 cache
```

需要同时满足：
- `_cq_enabled = True` ✅（`enable_cq_kv_cache` 已设置）
- `use_cache = True` ❌（`_model_call` 没有传递！）
- `past_key_values = None` ✅（首次调用）

**问题所在**（修复前的 `lm_eval/models/huggingface.py` 第 993 行）：

```python
def _model_call(self, inps, attn_mask=None, labels=None):
    # ...
    return self.model(inps).logits  # ❌ 没有 use_cache=True
```

由于缺少 `use_cache=True`，条件不满足，量化 cache 永远不会被创建和使用！

## 🔧 修复内容

### 修改的文件

**文件**：`lm_eval/models/huggingface.py`  
**位置**：第 993 行  
**类型**：单行修改

### 修改对比

```diff
  assert self.AUTO_MODEL_CLASS in (
      transformers.AutoModelForCausalLM,
      transformers.AutoModelForVision2Seq,
  )
- return self.model(inps).logits
+ # Enable cache for CQ quantization to take effect
+ return self.model(inps, use_cache=True).logits
```

### 修复后的流程

1. `_loglikelihood_tokens` 调用 `_model_call`
2. `_model_call` 调用 `model(inps, use_cache=True)`
3. 触发 `patched_forward`，检测到 `use_cache=True` 和 `past_key_values=None`
4. 创建 `QuantizedDynamicCache` 实例
5. KV cache 被量化存储（使用 codebook）
6. 在 attention 计算时反量化使用
7. **精度受到量化影响，会有所下降**

## 🔍 技术细节

### 为什么每个 batch 都创建新 cache？

在 `_loglikelihood_tokens` 中：
- 每个 batch 包含不同长度、不同内容的序列
- 它们之间不应该共享 KV cache
- 每次调用 `_model_call` 时：
  - `past_key_values=None`（我们没有传递旧 cache）
  - `use_cache=True`（新修复添加）
  - → 创建新的 `QuantizedDynamicCache`
- **这是正确的行为！**

### Seq2Seq 模型怎么办？

当前修复针对 Causal LM（如 Llama）。如果你使用 Seq2Seq 模型（如 T5），可能需要在第 985-987 行做类似修复。但由于 CQ 论文主要针对 decoder-only 模型，暂时不需要处理。

### 为什么 generation 没问题？

`_model_generate` 方法（第 1029 行）已经正确传递了 `use_cache=True`：

```python
return self.model.generate(
    input_ids=context,
    max_length=max_length,
    use_cache=True,  # ✅ 已经有了
    **generation_kwargs,
)
```

所以 generation 任务中 CQ 一直是正常工作的，只是 loglikelihood 评估有问题。
