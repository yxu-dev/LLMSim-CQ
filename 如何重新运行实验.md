## 🔧 重要参数说明

### 必需参数
- `--model hf`: 使用 HuggingFace 模型
- `--model_args pretrained=...`: 模型名称
- `--tasks`: 评测任务
- `--device`: GPU 设备

### CQ 量化专用参数
- `cq_codebook_dir=/path/to/codebook`: 启用 CQ 量化（**必须在 model_args 中**）
- 不加此参数 = Baseline（无量化）

### 可选参数
- `--batch_size 1`: 批次大小（建议保持 1，量化 cache 会占用更多内存）
- `--apply_chat_template`: 使用聊天模板（某些任务需要）
- `--fewshot_as_multiturn`: Few-shot 作为多轮对话
- `--limit N`: 只测试前 N 个样本（用于快速测试）
- `--output_path`: 结果保存路径

### 对比多个结果
创建对比脚本：

```bash
cat > compare_results.py << 'EOF'
import json
import sys

def load_result(path, name):
    with open(path) as f:
        data = json.load(f)
    return name, data

def compare_results(configs):
    """configs: [(path, name), ...]"""
    results = [load_result(p, n) for p, n in configs]
    
    # 获取所有任务
    tasks = list(results[0][1]['results'].keys())
    
    print(f"{'任务':<20} ", end="")
    for name, _ in results:
        print(f"{name:<15} ", end="")
    print()
    print("-" * (20 + 15 * len(results)))
    
    for task in tasks:
        print(f"{task:<20} ", end="")
        for name, data in results:
            acc = data['results'][task].get('acc,none', 
                  data['results'][task].get('acc_norm,none', 'N/A'))
            if isinstance(acc, float):
                print(f"{acc*100:>6.2f}%       ", end="")
            else:
                print(f"{str(acc):<15} ", end="")
        print()

if __name__ == "__main__":
    configs = [
        ("results/llama-3.1-8b/baseline_comprehensive.json", "Baseline"),
        ("results/llama-3.1-8b/cq_4c8b_comprehensive.json", "CQ-4c8b"),
        ("results/llama-3.1-8b/cq_8c8b_comprehensive.json", "CQ-8c8b"),
    ]
    compare_results(configs)
EOF

python compare_results.py
```

## ⚠️ 常见问题

### Q1: OOM (Out of Memory) 错误
**解决方案**：
- 降低 batch_size（已经是 1 了就没法降了）
- 减少 max_length（如果支持）
- 使用显存更大的 GPU
- 尝试 8c8b 配置（量化更温和，内存占用稍高）

### Q2: 运行很慢
**原因**：
- 量化/反量化有额外计算开销
- KV cache 操作频繁

**解决方案**：
- 这是正常的，CQ 量化会略微降低速度
- 可以先用 `--limit 50` 快速测试

### Q3: 结果还是没有差异
**检查清单**：
1. 是否在正确的 Python 环境？
2. 是否重启了 Python 进程（重新加载修改的代码）？
3. 运行 `./verify_cq_fix.sh` 确认修复生效
4. 检查 codebook 路径是否正确

### Q4: 不同任务的精度下降不一致
**这是正常的！**
- 不同任务对量化的敏感度不同
- 通常：数学推理 > 常识推理 > 语言理解
- 4c8b 比 8c8b 精度下降更多（更激进的量化）
